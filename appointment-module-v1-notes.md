# Bridge Appointment Module V1 Notes

## Implementation Summary

- Reused the existing Bridge appointment source-of-truth architecture instead of creating a separate calendar engine.
- Extended the appointment model with location type, meeting URL, timezone, all-day flag, related entity fields, cancellation metadata, custom type label, RSVP tokens, participant required/optional state, and invitation timestamps.
- Added a Supabase migration: `supabase/migrations/202605130001_appointment_module_v1.sql`.
- Added a public RSVP route: `/appointment-rsvp/:token`.
- Added RSVP RPC helpers so invitees can respond with opaque RSVP tokens without exposing the appointment tables directly.

## UI / UX

- Refined the existing Create Appointment modal in the Agent Pipeline calendar/workspace.
- Added structured sections:
  - Appointment Details
  - Date & Time
  - Workflow Link
  - Purpose / preparation
  - Availability checks
  - Participants
  - Notifications
  - Outcome / follow-up
- Calendar cards now show RSVP summary counts.
- Appointment detail actions include copy link, resend invite, cancel appointment, calendar invite actions, and open related record.

## Email / ICS

- Appointment emails now support RSVP action links:
  - Accept
  - Decline
  - Request Reschedule
- Appointment emails can include `.ics` calendar attachments.
- Sender preference is `RESEND_APPOINTMENTS_FROM_EMAIL`, falling back to `RESEND_FROM_EMAIL`, then `Bridge Appointments <appointments@bridge.co.za>`.

## Activity / Workflow

- Existing create/update/cancel paths continue writing lead activity and transaction events where related IDs are available.
- RSVP submissions record an `appointment_notification_events` entry when that table exists.

## Permissions

- The migration preserves authenticated appointment access through the existing organisation assignment permission helpers.
- Anonymous RSVP is handled through security-definer RPC functions using opaque tokens rather than broad table access.

## Known Gaps

- This does not add Gmail/Outlook account sync.
- Recurring appointments and advanced reminder scheduling are intentionally not implemented in V1.
- The `.ics` attachment generated by the edge email handler uses the appointment date/time payload; the richer frontend ICS generator remains available for manual download.
