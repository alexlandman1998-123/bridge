# Conveyancer productisation — P9 guided experience

P9 turns the P1–P8 capability set into a calm, guided conveyancer workspace. It changes presentation and interaction only; canonical matter facts, approvals, evidence and orchestration still pass through their existing guarded contracts.

## Delivered

- One plainly labelled next action with one primary button.
- Immediate work first: decisions, ready work and blockers are visible; waiting and later work are progressively disclosed.
- Plain-language queue labels and counts: ready, decisions, needs help and waiting.
- Legal-review safety: review items open the review workspace and cannot be completed from the summary card.
- Evidence gaps go directly to documents; blocked/resumed/waiting decisions still require an explicit human reason.
- Reminder, document, signing and provider status moved behind an optional systems disclosure.
- Every unavailable or stopped integration explains the manual continuity path.
- Technical plan IDs, revisions and receipts moved into audit details.
- Semantic buttons, lists and disclosures, labelled reason input, focus treatment, keyboard operation and live status messaging.
- Responsive two-column-to-single-column presentation using the existing design system.

## Acceptance

The conveyancer should be able to answer four questions without training: What should I do next? What needs my decision? What are we waiting for? Can I continue if automation is unavailable? The answer to the last question is always yes.

P9 adds no database migration and no new mutation path.
