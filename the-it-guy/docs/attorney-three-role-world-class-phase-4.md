# Attorney three-role world-class programme — Phase 4

Phase 4 gives the bond attorney a bank-instruction-led command centre on the shared matter workspace.

## Delivered

- Six operational domains: bank instruction and approval, bank conditions, drafting/signing, approval to lodge, guarantees/transfer coordination, and registration close-out.
- Existing bond workflow commands power the primary next action; Phase 4 does not create a competing workflow engine.
- Missing data, document, signature, and linked-handoff metrics are visible at a glance.
- Transfer dependencies are read-only source signals. The bond team can request or escalate a handoff from its own lane but cannot mutate transfer or cancellation work.
- Bond-lane permissions determine whether controls are executable or shown in coordination-only mode.

## Exit evidence

Run `npm run test:attorney-three-role-phase4`. It executes the Phase 0–3 regression chain, bond cockpit model tests, UI wiring checks, and existing workflow command verification.
